Student Survey Analytics

Messy Survey Data Cleaning & ML Imputation

A messy student survey export became a structured, privacy-aware dataset through two connected workflows: deterministic data cleaning and Random Forest imputation for sparse missing responses.

Project Arc

The dataset was small, but the data-quality problem was real: open-ended demographic fields, inconsistent category labels, mixed numeric text, and missing item-level survey answers.

  • Challenge Convert raw student survey responses into a consistent table without exposing row-level personal data.
  • Method Use rule-based cleaning for fields that needed deterministic logic, then use model-based imputation where missingness appeared across survey items.
  • Output Produce a cleaned 52-column analysis table and a documented schema that can be reviewed without the private workbook.
  • Use Case Prepare survey data for downstream student-support analysis while preserving as many valid responses as possible.

Impact Highlights

The strongest parts of the project are not about dataset size. They are about making messy human-entered data usable, auditable, and ready for modeling.

From messy text to reliable fields

Gender, race/ethnicity, relationship status, work hours, and caregiving responses were standardized into consistent analytical variables.

Missing values handled with ML

Seventeen Likert-scale fields were imputed with item-specific Random Forest models instead of dropping valid student responses.

Public story without private data

The repository explains the pipeline and table structure while keeping sensitive row-level survey answers out of the public version.

Two Core Methods

Method 01

Data Cleaning Pipeline

Built a deterministic ETL workflow that converted inconsistent free-text and categorical survey responses into repeatable analytical features.

  • Standardized gender, race/ethnicity, relationship status, work hours, and caregiving fields.
  • Parsed messy work-hour responses such as ranges and text into numeric weekly-hour values.
  • Converted multi-racial responses into one-hot encoded race indicators while preserving a primary cleaned category.
  • Produced a 52-column cleaned table from the original 47-column survey export.
Method 02

Machine Learning Imputation

Trained one Random Forest classifier per survey item with missing values, using cleaned demographic and caregiving fields as predictors.

  • Targeted 17 sparse missing Likert-scale fields across education and personal-experience questions.
  • Encoded categorical predictors with label encoding and filled numeric predictor gaps with medians.
  • Predicted missing answers only where enough training labels existed for that item.
  • Preserved the full 165-response analytical sample for later subgroup analysis.

Table Structure

The table begins as a Google Forms-style export and becomes a modeling-ready dataset after cleaning. Because the sample is small and privacy-sensitive, the public version documents field-level metadata rather than row-level responses.

1 timestamp field

Survey submission time retained for auditability but not used as a modeling feature.

6 profile fields

Birth year, gender, race/ethnicity, relationship status, work hours, and care work.

20 education items

Likert-scale responses about college access, anxiety, resources, and academic support.

20 personal-experience items

Likert-scale responses about trust, wellbeing, social experience, and daily constraints.

Project Takeaway

The project shows a practical data-science workflow for small, high-context survey data: clean the fields that require transparent rules, use machine learning only where prediction is appropriate, and document the table structure so the work can be reviewed without revealing private responses.
